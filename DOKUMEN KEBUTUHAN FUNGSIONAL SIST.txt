# DOKUMEN KEBUTUHAN FUNGSIONAL SISTEM

**Sistem Manajemen Usaha & Keuangan Terintegrasi**

---

## A. Gambaran Umum Sistem

### A.1 Latar Belakang

Sistem ini dirancang untuk membantu pemilik usaha dalam:

* Mengelola operasional usaha
* Mengelola keuangan usaha secara real-time
* Mengelola modal investor
* Memantau performa bisnis
* Melakukan proyeksi keuntungan
* Menghitung estimasi balik modal (Break Even Point)

Pada tahap awal, sistem digunakan untuk:

* Usaha warnet + mini cafe
* Pendanaan dari investor
* Pemakaian internal

Namun arsitektur sistem dirancang agar:

* Dapat digunakan untuk berbagai jenis usaha
* Dapat dikembangkan menjadi platform SaaS multi-tenant

---

## B. Tujuan Sistem

Sistem ini bertujuan untuk:

1. Menyediakan pencatatan keuangan yang terstruktur
2. Memberikan laporan keuangan secara real-time
3. Mengelola hubungan antara pemilik usaha dan investor
4. Menghitung keuntungan, kerugian, dan ROI
5. Membantu pengambilan keputusan bisnis berbasis data
6. Meminimalkan kesalahan pencatatan manual
7. Menyediakan transparansi kepada investor

---

## C. Target Pengguna Sistem

Sistem dirancang untuk digunakan oleh beberapa jenis pengguna:

* Pemilik Usaha (Owner)
* Investor
* Admin Operasional
* Kasir
* Akuntan / Keuangan

---

# 1. Modul Manajemen Pengguna dan Akses

### 1.1 Manajemen Autentikasi

Sistem harus mendukung:

* Login berbasis email & password
* Autentikasi berbasis JWT
* Manajemen sesi pengguna
* Manajemen refresh token
* Logout sistem

---

### 1.2 Role Management

Sistem harus mendukung pengelolaan hak akses berbasis role:

Role utama:

1. Owner
2. Admin Operasional
3. Kasir
4. Investor
5. Akuntan (opsional)

---

### 1.3 Hak Akses Per Role

#### Owner

Memiliki akses penuh terhadap:

* Semua data keuangan
* Laporan lengkap
* Manajemen investor
* Konfigurasi sistem
* Approval transaksi tertentu
* Analisis dan proyeksi

#### Investor

Hanya dapat mengakses:

* Dashboard investor
* Laporan read-only
* Nilai ROI
* Progress modal
* Perkiraan keuntungan

#### Admin Operasional / Kasir

Dapat melakukan:

* Input penjualan
* Input pembelian
* Input biaya
* Tidak dapat melihat data investor

#### Akuntan

* Mengelola laporan keuangan
* Verifikasi transaksi
* Koreksi pencatatan

---

### 1.4 Keamanan Akses

Sistem wajib menyediakan:

* Role Based Access Control
* Pembatasan akses API
* Audit log aktivitas
* Pencatatan histori perubahan data

---

# 2. Modul Manajemen Modal Usaha

Modul ini menjadi inti dari hubungan pemilik usaha dan investor.

---

### 2.1 Manajemen Data Investor

Sistem harus dapat menyimpan data investor secara lengkap:

* Identitas investor
* Nilai investasi
* Tanggal masuk modal
* Jenis investasi
* Persentase kepemilikan
* Riwayat transaksi modal

---

### 2.2 Pengelolaan Modal

Fitur yang harus tersedia:

* Pencatatan setoran modal awal
* Penambahan modal
* Penarikan modal
* Perubahan kepemilikan
* Perhitungan otomatis proporsi saham/modal

---

### 2.3 Skema Bagi Hasil

Sistem harus fleksibel mendukung:

* Skema profit sharing
* Skema dividen tetap
* Skema proporsional modal
* Skema custom per investor

---

## D. Spesifikasi Teknis

Arsitektur yang digunakan:
* backend dan frontend terpisah
* Backend: NestJS
* Frontend: Next.js + Tailwind
* Database: PostgreSQL
* API: REST API
* Auth: JWT
* Reporting: PDF & Excel
* Gunakan GIT
